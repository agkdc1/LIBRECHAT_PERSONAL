# LibreChat on GCP with Vertex AI

A personal, cloud-native [LibreChat](https://github.com/danny-avila/LibreChat) deployment on Google Cloud Platform, powered by Vertex AI (Gemini models). The entire infrastructure is defined in Terraform and designed to run within GCP's free tier.

## Architecture

```
Browser
  |
  v
Cloudflare Access (email-verified, owner-only)
  |
  v
Cloudflare Worker (rewrites Host header)
  |
  v
Cloud Run (us-central1) ---- VPC egress ----> MongoDB (GCE e2-micro)
  |                                              |
  |-- Vertex AI (Gemini via ADC)                 |-- Auth enabled + custom port
  |-- Secrets from Secret Manager                |-- Secrets from Secret Manager
  |-- Container from Artifact Registry           |-- Firewall: internal only
```

### How it works

1. **Cloudflare Access** gates the custom domain. Only the owner's email can pass (verified via one-time PIN). Everyone else is blocked before reaching GCP.

2. A **Cloudflare Worker** intercepts the request and rewrites the `Host` header from the custom domain to Cloud Run's `.run.app` hostname, so Cloud Run can route it correctly.

3. **Cloud Run** serves the LibreChat application. It pulls the container image from GitHub Container Registry via an **Artifact Registry** remote repository (Cloud Run only supports GCR, Artifact Registry, and Docker Hub natively). The service account has `aiplatform.user` role for Vertex AI access via Application Default Credentials — no API keys needed.

4. **MongoDB** runs on a free-tier `e2-micro` VM in the same VPC. Cloud Run connects to it over a private network (VPC egress). MongoDB is configured with a random strong password, a random high port, and an auth keyfile — all generated by Terraform and stored in **Secret Manager**. The MongoDB port is only accessible from Cloud Run's subnet via firewall rules; SSH access is restricted to IAP tunneling.

5. A **GCS bucket** is provisioned for static frontend assets and file uploads.

## Prerequisites

- [Terraform](https://developer.hashicorp.com/terraform/install) >= 1.5
- [gcloud CLI](https://cloud.google.com/sdk/docs/install) authenticated (`gcloud auth application-default login`)
- A GCP billing account
- A domain managed by Cloudflare
- A [Cloudflare API token](https://dash.cloudflare.com/profile/api-tokens) with permissions:
  - Zone > DNS > Edit
  - Account > Cloudflare Tunnel > Edit
  - Account > Access: Apps and Policies > Edit
  - Account > Zero Trust > Edit
  - Account > Workers Scripts > Edit

## Setup

### 1. Clone and configure

```bash
git clone https://github.com/agkdc1/LIBRECHAT_PERSONAL.git
cd LIBRECHAT_PERSONAL
cp terraform.tfvars.example terraform.tfvars
```

Edit `terraform.tfvars`:

```hcl
project_id      = "my-librechat-project"    # globally unique GCP project ID
billing_account = "XXXXXX-XXXXXX-XXXXXX"    # GCP billing account ID
owner_email     = "you@gmail.com"           # only this email can access
custom_domain   = "chat.example.com"        # subdomain on your Cloudflare zone
```

### 2. Deploy

```bash
terraform init
terraform apply -var="cloudflare_api_token=YOUR_CF_TOKEN"
```

Terraform will:
- Create a GCP project and enable APIs
- Generate random MongoDB credentials, JWT secrets, and encryption keys
- Store everything in Secret Manager
- Provision the VPC, firewall rules, and MongoDB VM
- Deploy LibreChat on Cloud Run
- Set up Cloudflare DNS, Worker, and Access policy

### 3. First login

1. Visit your custom domain
2. Cloudflare Access will prompt for your email and send a verification PIN
3. After verification, LibreChat's registration page appears
4. Create your account

After registering, disable new registrations by setting `ALLOW_REGISTRATION = "false"` in `cloudrun.tf` and re-applying.

## Gemini Models

Models are configured in `librechat.yaml`:

```yaml
endpoints:
  google:
    models:
      default:
        - "gemini-3.1-pro-preview"
        - "gemini-3.1-flash-preview"
        - "gemini-3.0-deep-think"
    modelDisplayLabel: "Vertex AI"
    maxContextTokens: 1048576
    maxOutputTokens: 65536
```

To add or remove models, edit this file and run `terraform apply` — the config is passed to Cloud Run as a base64-encoded environment variable.

## Security

- **Access control**: Cloudflare Access blocks all unauthenticated requests at the edge. Only the configured email can reach the application.
- **MongoDB**: Listens on a random high port, requires password auth + keyfile, and is only reachable from Cloud Run's subnet via VPC firewall rules. SSH access is restricted to GCP IAP tunneling.
- **Secrets**: All credentials are randomly generated by Terraform, stored in GCP Secret Manager, and injected into Cloud Run as secret environment variables. Nothing is hardcoded in code.
- **Network**: Cloud Run connects to MongoDB via private VPC egress. MongoDB's port is never exposed to the internet.
- **No sensitive data in git**: Email, domain, billing account, project ID, and all tokens are in `terraform.tfvars` (gitignored) or passed via CLI flags.

## Operations

```bash
# SSH into MongoDB VM
gcloud compute ssh mongodb --zone=us-central1-a --tunnel-through-iap --project=PROJECT_ID

# View Cloud Run logs
gcloud run services logs read librechat --region=us-central1 --project=PROJECT_ID

# Check MongoDB startup progress
gcloud compute ssh mongodb --zone=us-central1-a --tunnel-through-iap --project=PROJECT_ID \
  -- sudo journalctl -u google-startup-scripts -f

# Destroy everything
terraform destroy -var="cloudflare_api_token=YOUR_CF_TOKEN"
```
